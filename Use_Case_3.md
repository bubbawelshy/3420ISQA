<b>Title:</b> Software Policy Update
<br><br>
<b>Primary Actor:</b> Corporate Manager
<br><br>
<b>Goal in Context:</b> The manager is able to make updates to an existing policy regarding a specific Software Package.
<br><br>
<b>Stakeholders:</b> 
<ul>
<li><i>Corporate Manager:</i> To reevaluate existing policy documents in regards to a specific Software Package.</li>
</ul>
<b>Preconditions:</b>
<ul>
<li>There is existing information about a Software Packages Licenses and Vulnerabilities in a database.</li>
<li>The Corporate Manager has access to view existing policies.</li> 
</ul>
<b>Main Success Scenario:</b> The Corporate Manager successfully updates the policy and stores the updated policy into the database.
<br><br>
<b>Failed End Connections:</b> There is not an existing policy for the Corporate Manager to view and update.
<br><br>
<b>Trigger:</b> When new information is presented and the policy needs to be updated.
